**SPA AI is a developer agent that installs machine-level halts for the failures developers currently absorb through vigilance.**

Toyoda Sakichi invented the first loom that stops itself when a thread breaks. Before his 1924 Type G, a weaver — often his own mother — stood vigilant through the night, catching defects by attention. After, the loom caught them. The weaver became free to judge, improve, extend — to do what only a human can do. The machine took the vigilance; the human kept the mind.

A century later, software developers stand in the same position: staying late to review PRs for formatting drift, debugging silent failures that a typed return could have prevented, absorbing deprecation warnings a CI step could have halted on. The industry calls this *"engineering attention."* We call it vigilance — the same burden Sakichi's loom removed.

SPA AI scans a git repository, finds the halts that are missing, and installs them as pull requests. Pre-commit hooks. Type stubs. Contract tests. CODEOWNERS routing. Each PR cites the specific Jidoka principle it derives from. The developer reviews, merges, and moves on. Next time the thread breaks, the loom catches it.

*A broken thread must stop the loom. The loom serves the weaver. Never blame the human — build the loom that should have caught it.*
